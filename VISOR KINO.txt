1) Decisiones clave de arquitectura (recomendación)
Opción A (rápida y robusta)

Web app + Base de datos + Almacenamiento en nube

Backend: Node/Express o PHP (Laravel) o Python (Flask)

DB: MySQL/PostgreSQL

Storage: Google Drive / Dropbox / S3 (ideal S3-compatible)

Recomendación práctica:
Si quieres cero lío con permisos y alto control: S3 compatible (Wasabi/Cloudflare R2/AWS S3).
Si quieres aprovechar lo que ya usas: Google Drive, pero con reglas claras de enlaces.

6) Descargas (ZIP) y rendimiento
6.1 ZIP bajo demanda + caché

Cuando el cliente pide “Descargar Fotos”, el backend:

Reúne las imágenes (de Drive o storage)

Crea ZIP

Guarda ZIP cacheado por 24h o hasta cambio de media

Devuelve link de descarga

6.2 Estrategia si es Drive

Dos caminos:

Camino simple: el botón abre la carpeta (menos pro).

Camino pro: el servidor descarga temporalmente desde Drive y genera ZIP.
(Esto requiere credenciales y es más trabajo, pero queda “premium”.)

Recomendación para 4.000 SKUs: ZIP bajo demanda con caché.
